# 20 — UX Design

## 1. 사용자 플로우 다이어그램

```
┌─────────────────────────────────────────────────────────────────┐
│                         학습 여정 흐름도                          │
└─────────────────────────────────────────────────────────────────┘

[시작]
  ↓
[대시보드] (#/)
  │
  ├─→ Stage 1 카드 클릭
  │     ↓
  │   [모듈 목록] (#/stage/1)
  │     ↓
  │   [모듈 선택 1-1 ~ 1-8]
  │     │
  │     ├─→ 모듈 1-1 ~ 1-7: 일반 학습
  │     │     ↓
  │     │   [학습 화면] (#/module/{id})
  │     │   ├─ 패턴 A: 읽기 카드 (Step-by-Step)
  │     │   ├─ 패턴 B: 선택 퀴즈 (정답/오답 피드백)
  │     │   └─ 패턴 C: 다단계 실습 (비교 화면)
  │     │     ↓
  │     │   [결과 화면] (#/module/{id}/result)
  │     │   ├─ 완료 메시지
  │     │   └─ 다음 모듈로 / 모듈 목록으로
  │     │
  │     └─→ 모듈 1-8: 종합평가
  │           ↓
  │         [결과 화면] (#/module/1-8/result)
  │         ├─ 점수 표시
  │         ├─ [합격 70%↑] → Stage 2 해제 + 대시보드로 이동
  │         └─ [불합격 <70%] → "다시 도전" + 취약 영역 안내
  │
  │
  └─→ Stage 2 카드 클릭 (Stage 1 완료 후 해제)
      └─ [모듈 목록] (#/stage/2)
         └─ [모듈 선택 2-1 ~ 2-8]
            └─ [학습 → 결과] (반복)
```

**주요 분기점**:
1. **Stage 1 미완료**: Stage 2 카드 잠금 상태, "Stage 1 완료 필요" 메시지
2. **모듈 불합격**: "다시 도전" 버튼 → 같은 모듈 재학습
3. **모듈 합격**: "다음 모듈로" → 다음 모듈 학습 화면

---

## 2. 화면 목록

| # | 화면명 | URL | 목적 | 주요 컴포넌트 |
|---|--------|-----|------|---------------|
| S1 | 대시보드 | `#/` | 학습 진도 현황 확인, Stage 선택 | Stage 카드 (2개), 전체 진도바, 잠금 상태 표시 |
| S2 | 모듈 목록 | `#/stage/{id}` | Stage별 모듈 목록 확인, 모듈 선택 | Stage 제목, 모듈 카드 (8개), 뒤로가기, 필터/정렬 옵션 |
| S3 | 학습 화면 | `#/module/{id}` | 모듈 학습 (패턴 A/B/C) | 모듈 제목, Step 카운터, 진행 프로그래스바, Step 콘텐츠, 네비게이션 |
| S4 | 결과 화면 | `#/module/{id}/result` | 학습 완료, 성과 표시, 다음 액션 | 완료 메시지, 점수/정답률, 다음 버튼, 취약점 분석 |

## 5. 상태별 화면 명세

### 5.1 로딩 상태

**정의**: 서버에서 데이터를 불러오는 중인 상태

**표시 방식**:
- 스켈레톤 로딩 (Skeleton Loading): 회색 박스로 콘텐츠 영역을 표현
- 또는 로딩 스피너 (Spinner): 중앙에 회전하는 원형 로더

**적용 화면**:
- S1 (대시보드): Stage 카드 로딩
- S2 (모듈 목록): 모듈 카드 목록 로딩
- S3 (학습 화면): Step 콘텐츠 로딩
- S4 (결과 화면): 결과 데이터 로딩

---

### 5.2 빈 상태 (Empty State)

**정의**: 표시할 데이터가 없는 상태 (정상 운영 중에는 발생하지 않으나, 예외 상황에 대비)

**표시 방식**:
- 아이콘 + 메시지 + CTA 버튼
- 예: "준비 중인 모듈입니다" 메시지 + "모듈 목록으로" 버튼

**시나리오**:
- Stage 2가 아직 해제되지 않음 → "Stage 1 완료 후 해제" 메시지
- 모듈이 준비되지 않음 (개발 단계) → "준비 중" 메시지

---

### 5.3 에러 상태

**정의**: 네트워크 오류, 서버 오류, 데이터 불일치 등으로 작업 완료 불가

**표시 방식**:
- 경고 아이콘 + 에러 메시지 + 재시도 버튼

**에러 메시지 유형**:

| 상황 | 메시지 | 버튼 |
|------|--------|------|
| 네트워크 오류 | "네트워크 연결을 확인해주세요" | [새로고침], [뒤로가기] |
| 서버 오류 | "서버 오류가 발생했습니다" | [새로고침], [메인으로] |
| 타이ム아웃 | "요청 시간이 초과되었습니다" | [재시도], [뒤로가기] |
| 데이터 오류 | "모듈 데이터가 손상되었습니다" | [메인으로], [고객지원] |

## 6. 라우팅 및 네비게이션

### 6.1 Hash-based SPA 라우트

| 라우트 | 화면 | 이전 화면 | 다음 화면 |
|--------|------|-----------|----------|
| `#/` | 대시보드 (S1) | — | S2 (#/stage/{id}) |
| `#/stage/{id}` | 모듈 목록 (S2) | S1 | S3 (#/module/{id}) |
| `#/module/{id}` | 학습 화면 (S3) | S2 | S4 (#/module/{id}/result) |
| `#/module/{id}/result` | 결과 화면 (S4) | S3 | S2 또는 S1 |

**뒤로가기 동작**:
- S2에서 뒤로: `#/` (대시보드)
- S3에서 뒤로: `#/stage/{id}` (모듈 목록)
- S4에서 뒤로: `#/stage/{id}` (모듈 목록) 또는 `#/` (대시보드)

## 8. 접근성 및 UX 원칙

### 8.1 색상 및 대비

- **배경**: 흰색 (#FFFFFF) 또는 밝은 회색 (#F5F5F5)

## 정리: UX 설계 원칙

1. **선택 기반만으로 전체 학습 피드백 제공**: 쿨링(코칭) 피드백, 정답/오답 해설, 비교 분석 제공
2. **진도 시각화**: Step 카운터 + 프로그래스바로 명확한 학습 위치 표시
3. **단계적 네비게이션**: 이전/다음 버튼으로 선형 진행, 뒤로가기로 유연한 복귀
4. **상태별 맥락**: 로딩/에러/완료 등 모든 상태를 명시적으로 표시
5. **결과 피드백**: 점수, 정답률, 취약점, 다음 액션을 명확하게 제시
6. **모바일 미대응**: 데스크톱 1024px 기준 설계로 단순화
