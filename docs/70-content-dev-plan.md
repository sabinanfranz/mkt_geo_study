# 70 — 콘텐츠 개발 계획 (Phase 2)

## 개요

### 목표
Phase 2는 GEO 멘토링 학습 시스템의 **비주얼 강화** 및 **Stage 3~6 확장**을 수행한다.

### 현재 상태 (Phase 1 완료)
- **2 Stages** / 16 모듈 / 59 스텝 / 179 옵션
- US-001~025 전체 25개 유저스토리 완료
- 28개 자동 테스트(pytest) 통과
- API 9개 엔드포인트 + SPA 5개 화면

### 목표 상태 (Phase 2 완료 후)
- **6 Stages** / 44 모듈 / 174 스텝 / 539 옵션
- Stage 1~2 기존 콘텐츠에 비주얼 요소 14개 보강
- CSS 비주얼 컴포넌트 6종 신규

## 1. 비주얼 컴포넌트 시스템

### 기술 접근법

| 항목 | 설명 |
|------|------|
| 렌더링 방식 | `marked.js`가 마크다운 내 인라인 HTML을 그대로 렌더링 |
| 코드 변경 | **CSS만 추가** — `app.js` 수정 불필요 |
| 이미지 의존 | 없음. **CSS + Unicode/Emoji**로 모든 비주얼 처리 |
| 다크모드 | `[data-theme="dark"]` CSS 오버라이드 전 컴포넌트 포함 |
| 삽입 위치 | `content_md` 및 `extension_md` 문자열 내 HTML 블록 |

### 6가지 컴포넌트

| # | 컴포넌트 | CSS 클래스 | 용도 | 주 사용 위치 |
|---|---------|-----------|------|-------------|
| 1 | 브라우저 목업 | `.browser-mockup` | 봇이 보는 웹페이지 시뮬레이션 (주소창 + 콘텐츠 + present/missing 상태) | M1-1, M1-3, M1-7, M2-3, M3-1 |
| 2 | Before/After 비교 카드 | `.compare-cards` | 현재 상태 vs 개선 후 나란히 비교 | M1-2, M1-6, M2-2, M2-5, M3-2 |
| 3 | 다이어그램 박스 | `.diagram-box` | RAG 파이프라인, KPI 계층 등 개념 시각화 | M1-3, M1-4, M1-5, M2-1, M2-6 |
| 4 | 콜아웃 박스 | `.callout` | 팁/경고/핵심/힌트 4종 (아이콘 + 색상 구분) | 전 모듈 reading 스텝 |
| 5 | 코드 예시 박스 | `.code-example` | JSON-LD, HTML 코드 스니펫 표시 | M2-4, M3-1~M3-6, M4-5, M5-3 |
| 6 | 계층 구조 박스 | `.hierarchy-box` | H1→H2→H3 문서 구조 시각화 | M2-5, M3-2 |

### 콜아웃 4종 변형

| 타입 | 아이콘 | 색상 | 용도 |
|------|--------|------|------|
| `.callout.tip` | 💡 | 파랑 | 실용적 팁 |
| `.callout.warning` | ⚠️ | 노랑 | 주의사항/흔한 실수 |
| `.callout.key-point` | 🎯 | 초록 | 핵심 포인트 |
| `.callout.hint` | 🔍 | 연초록 | 심화 힌트 |
