## 0. Executive Summary

### 전략 한 줄
**b2b.fastcampus.co.kr을 AI 검색(ChatGPT Search + Google AI Overviews/AI Mode)에서 더 자주 인용되게 만들기 위해, 상단 정답문장 + 근거블록 + 팬아웃 콘텐츠 + 측정 루프를 90일간 반복 실행한다.**

### 4대 실행 레버
1. **Answer-first**: 페이지 상단 2~6문장으로 정의/대상/제공형태/산출물/도입조건을 고정
2. **Proof-first**: 통계/인용/외부 출처(3~7개) 블록을 Answer 바로 아래에 표준화
3. **Fan-out 클러스터**: 허브 7개 + 스포크 60개 1차 발행으로 하위 질의 커버리지 확대
4. **측정 실험 루프**: Top 50 질문 세트로 주간 인용률/커버리지 추적 및 4주 단위 실험

### 30/60/90일 마일스톤
- **30일**: 상위 10 랜딩 Answer-first/Proof-first 적용 + 측정 체계 세팅 (**28 tickets**)
- **60일**: 허브 7개 구축 + 스포크 60개 1차 발행 + 자산 표준화 (**20 tickets**)
- **90일**: 구조화 데이터 정착 + 내부링크 그래프 재설계 + 외부 권위 확보 (**11 tickets**)

**총 백로그**: ~59 tickets, P0:28 / P1:20 / P2:11

## 1. 현재 상태 진단

### 1.1 크롤링 기술 현황

#### robots.txt & sitemap.xml 상태
- **robots.txt**: 404 (Not Found) → 모든 봇 허용 (의도적 설정 필요)
- **sitemap.xml**: 404 (Not Found) → 생성 필요
- **현재 상태**: 검색 봇이 포함/제외 정책을 명확히 알 수 없음
- **영향**: ChatGPT Search(OAI-SearchBot), Google Bot 모두 무제약 크롤 → 허용 정책 명문화 필요

#### JavaScript 렌더링 현황
- **총 크롤 페이지**: 25개
- **CSR(클라이언트 사이드 렌더링)**: 22개 (88%)
- **SSR(서버 사이드 렌더링)**: 3개 (12%)
- **문제점**: 22/25 페이지는 Next.js 기반 SPA로, 검색 봇이 JS 렌더링 미지원 시 콘텐츠 읽기 불가능
- **크롤러 검증 방법**: Selenium 기반 JS 렌더링 크롤은 성공하나, 일반 HTTP 크롤은 실패 가능성 높음

#### 구조화 데이터 현황
- **JSON-LD 보유**: 25개 페이지 중 25개 모두 @type: Course 마크업 (100%)
- **문제점**:
  - `/contact`, `/contact_mktg` 등 실제로는 ContactPage여야 할 페이지도 Course 표기
  - 템플릿 기본값 오용으로 보임, 의도적 구조화 아님
- **필수 스키마 부재**:
  - BreadcrumbList: 0/25
  - Organization: 0/25 (필수)
  - Event: 0/25 (세미나 페이지 대상)
  - Article: 0/25 (인사이트 페이지 대상)

### 1.3 내부 링크 구조 분석

**정량 지표**:
- **총 내부 링크**: 1,028개 (25개 페이지)
- **페이지당 평균**: 41개
- **하지만 구조**: Flat (허브-스포크 아키텍처 전무)

**현재 구조의 문제점**:
1. **허브 부족**: 모든 페이지가 홈, 문의, 개인정보로만 연결
2. **서비스 페이지 고립**: service_custom, service_online 등이 서로 연결 안 됨
3. **사례 허브 미활용**: refer_customer가 다른 사례/관련 페이지로 연결 안 됨
4. **주제 전문성 신호 약함**: 관련 페이지들이 명시적으로 연결 안 됨

**필요한 개선**:
- Hub-Spoke 구조로 재설계 (허브 7개 중심)
- 관련 스포크 간 상호 링크
- Breadcrumb 구조화 데이터로 IA 명시

