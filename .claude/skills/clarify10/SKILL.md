# Clarify10 — 플랜 구체화 10문 10답

## Workflow

### 1. 컨텍스트 파악

현재 대화에서 사용자의 **플랜/기획 문서**를 식별한다.
플랜이 명시적 문서가 아니라 대화 흐름 속에 있을 수도 있다.
핵심 파악 대상:
- 무엇을 만들려는가 (산출물)
- 왜 만드는가 (목적/배경)
- 현재 결정된 것 vs 아직 모호한 것

### 2. 10대 차원에서 미결 항목 탐색

아래 **12개 후보 차원** 중 현재 플랜에서 **가장 모호하거나 미결정인 10개**를 선택한다.
이미 명확히 결정된 차원은 건너뛰고, 대신 다른 차원을 채운다.

| # | 차원 | 탐색 포인트 |
|---|---|---|
| S | **Scope** | 범위 안에 들어가는 것 / 빠지는 것 경계 |
| P | **Priority** | 무엇을 먼저, 무엇을 나중에 |
| A | **Architecture** | 기술 구조, 시스템 설계, 컴포넌트 분할 |
| D | **Delivery** | 산출물 형태, 포맷, 전달 방식 |
| U | **User/Audience** | 최종 사용자, 수혜자, 의사결정자 |
| C | **Constraints** | 제약 조건 (예산, 기술, 인력, 규제) |
| Q | **Quality** | 완성도 기준, 통과/불통과 판단 기준 |
| T | **Timeline** | 일정, 마일스톤, 데드라인 |
| R | **Risk** | 리스크, 실패 시나리오, 대비책 |
| I | **Integration** | 기존 시스템/프로세스/자산과의 연결 |
| E | **Evolution** | 반복/개선 주기, 버저닝, 유지보수 |
| X | **Experience** | UX, 인터랙션, 사용 흐름 |

### 3. 질문 생성 규칙

각 질문은 아래 포맷을 따른다:

```
**Q{N}. [{차원 태그}] {구체적 질문}**

| 옵션 | 설명 |
|---|---|
| A) {옵션A} | {1줄 설명} |
| B) {옵션B} | {1줄 설명} |
| C) {옵션C} | {1줄 설명} |

> 추천: **{옵션}** — {추천 근거 1문장}
```

질문 작성 원칙:
- 질문은 **닫힌 질문** (3개 중 택1)으로 구성하되, 사용자가 "D) 다른 의견"을 자유롭게 추가할 수 있다고 안내
- 옵션은 **상호 배타적**이고 **현실적으로 실행 가능**해야 한다
- 추천 옵션은 현재 컨텍스트 기반으로 판단하되, **왜 그 옵션인지** 근거를 1문장으로 명시
- 질문 순서: 상위 의사결정(Scope, Priority) → 구조적 결정(Architecture, Delivery) → 세부 결정(Quality, Timeline) 순

### 4. 출력 구조

```markdown
# Clarify10: {플랜 제목 요약}

> 현재 플랜에서 **결정이 필요한 10가지 핵심 질문**입니다.
> 각 질문에 A/B/C 중 선택하거나, D) 다른 의견을 자유롭게 적어주세요.

---

**Q1. [Scope] ...**
(위 포맷 반복 x 10)

---

## Quick Pick (한눈에 보기)

| Q | 차원 | 추천 | 요약 |
|---|---|---|---|
| 1 | Scope | B | ... |
| 2 | Priority | A | ... |
| ... | ... | ... | ... |
| 10 | ... | ... | ... |

> 위 추천안 전체를 수용하려면 **"추천대로"** 라고 답해주세요.
> 일부만 변경하려면 **"Q3=C, Q7=A"** 형태로 답해주세요.
```

### 5. 응답 처리

사용자가 답변하면:
- "추천대로" → 추천안 전체 반영, 플랜 업데이트 제안
- "Q3=C, Q7=A" → 해당 항목만 변경, 나머지는 추천 유지
- 개별 답변 → 해당 질문에 반영
- 추가 논의 필요 → 해당 차원에 대해 후속 질문 1~2개 추가

답변 반영 후, 구체화된 플랜을 **업데이트된 문서 형태**로 제시한다.
